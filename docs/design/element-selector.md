## 概述

Mimicry 的元素选择器系统提供三种入口、智能分析面板和选择器自愈机制，确保浏览器自动化中的元素定位既易用又稳定。

```

│                                                         │
│  入口层                                                  │
│  ┌───────────┐  ┌───────────────┐  ┌────────────────┐  │
│  │ 录制模式   │  │ Block配置面板  │  │ 智能分析面板    │  │
│  │ 自动生成   │  │ 手动编辑      │  │ 多策略候选     │  │
│  └─────┬─────┘  └──────┬────────┘  └───────┬────────┘  │
│        └───────────────┼────────────────────┘           │
│                        ▼                                │
│  分析层                                                  │
│  ┌─────────────────────────────────────────────────┐    │
│  │  选择器生成引擎                                    │    │
│  │  - 多策略生成 (text/id/data-*/role/class/xpath)  │    │
│  │  - 唯一性检测                                     │    │

## 三个触发入口

### 1. 录制模式自动生成

用户在录制模式下操作浏览器时，Recorder 模块自动捕获 DOM 事件并生成选择器：

```
用户操作 → DOM Event → Recorder 捕获 → 选择器引擎分析 → 生成 Block + 选择器
```

- 每次交互自动生成主选择器 + 多个备选选择器
- 录制结果直接存入 Block 的 `data.selector` 和 `data.selectorFallbacks`
- 录制完成后可在 Block 面板中回顾和调整

### 2. Block 配置面板手动编辑

在工作流编辑器的 Block 属性面板中：

```
┌─────────────────────────────────────┐
│  Click Block 配置                    │
│                                     │
│  选择器: [#submit-btn          ] 🎯  │
│                                     │
│  备选选择器:                         │
│  ┌─────────────────────────────┐    │
│  │ 1. text=Submit       ⭐ 92  │    │
│  │ 2. [data-testid=submit] 88  │    │
│  │ 3. .btn.btn-primary     71  │    │
│  │ 4. //button[@type=...   55  │    │
│  └─────────────────────────────┘    │
│                                     │
│  [🔍 测试] [🎯 重新选择] [📋 复制]   │
└─────────────────────────────────────┘
```

- 手动输入/修改选择器，支持 CSS / XPath / Playwright 语法
- 点击 🎯 按钮激活浏览器高亮模式，可视化重新选择
- 点击 🔍 测试按钮，在浏览器中高亮匹配元素并显示匹配数量

### 3. 智能分析面板

独立的分析面板，展示选择器的详细信息：

```

│                                          │

## 浏览器高亮模式交互

当用户激活元素选择器时，Python Sidecar 向浏览器注入高亮脚本：

### 交互流程

```
1. 用户点击 🎯 按钮
2. 前端 → Rust → Python: 调用 selector.startPicking()
3. Python 注入高亮 overlay 脚本到页面
4. 用户移动鼠标 → 元素高亮（蓝色边框 + 信息浮窗）
5. 浮窗显示: 标签名、尺寸、关键属性
6. 用户点击元素 → 锁定选择
7. Python 分析元素 → 返回候选选择器列表
8. 前端更新 Block 配置面板
9. 按 Esc 取消选择
```

### 高亮样式

```
┌─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─┐
│  ┌──────────────────────────┐       │
│  │  [button.btn-primary]    │ ◄── 蓝色边框 + 半透明遮罩
│  │  318 × 42               │       │
│  └──────────────────────────┘       │
│       ▲ 信息浮窗                     │
└─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─┘
```

## 智能分析面板设计

### 选择器生成策略

按优先级从高到低，为每个元素生成多个候选选择器：

| 优先级 | 策略 | 说明 | 稳定性基准分 |
|--------|------|------|-------------|
| 1 | `text=` | Playwright 文本定位 | 90 |
| 2 | `id` | HTML id 属性 | 95 |
| 3 | `data-testid` / `data-*` | 测试专用属性 | 95 |
| 4 | `role=` | ARIA 角色定位 | 85 |
| 5 | `class` 组合 | CSS 类名组合 | 60 |
| 6 | CSS Path | 完整 CSS 路径 | 40 |
| 7 | XPath | XPath 绝对路径 | 30 |

### 唯一性检测

对每个候选选择器在当前页面执行匹配，检查是否唯一命中目标元素：

```python
# Python Sidecar 中的唯一性检测
async def check_uniqueness(page, selector: str) -> dict:
    elements = await page.locator(selector).all()
    return {
        "selector": selector,
        "matchCount": len(elements),
        "isUnique": len(elements) == 1
    }
```

### 稳定性评分

综合以下因素计算 0-100 分：

| 因素 | 权重 | 说明 |
|------|------|------|
| 策略类型 | 40% | `id` > `text=` > `role=` > `class` > `xpath` |
| 唯一性 | 30% | 唯一匹配得满分，多匹配按比例扣分 |
| DOM 深度 | 15% | 路径越短越稳定 |
| 属性动态性 | 15% | 含随机字符串的属性扣分 |

## 快速操作

在智能分析面板中，用户选定元素后可一键生成常用 Block：

| 按钮 | 生成的 Block | 预填数据 |
|------|------------|---------|
| 📝 获取文本 | `data/GetText` | selector = 当前选择器 |
| 👆 点击 | `interaction/Click` | selector = 当前选择器 |
| ⌨️ 输入 | `interaction/Type` | selector = 当前选择器 |
| 📸 截图 | `data/Screenshot` | selector = 当前选择器（元素截图） |

生成的 Block 自动添加到画布当前位置，并连接到选中的前驱节点。

## 录制模式特殊处理

录制模式下，选择器生成需要额外考虑：

1. **实时性**：用户操作连续产生，每个事件需在 < 100ms 内生成选择器
2. **去抖动**：连续的 mousemove/scroll 事件合并处理
3. **上下文关联**：在 Loop Elements 循环中录制时，自动使用相对选择器 `{{$loop.current}} >> .child`
4. **iframe 感知**：在 iframe 内录制时，自动添加 frame 切换步骤
5. **智能合并**：连续在同一输入框的 keypress 事件合并为单个 Type Block

## 选择器自愈机制

执行工作流时，如果主选择器无法匹配元素，自动尝试备选选择器：

```
执行 Block
    │
    ▼
主选择器匹配? ──Yes──► 执行操作 ──► 完成
    │
    No
    ▼
遍历 selectorFallbacks
    │
    ├─ fallback[0] 匹配? ──Yes──► 执行操作 + 记录自愈日志
    │
    ├─ fallback[1] 匹配? ──Yes──► 执行操作 + 记录自愈日志
    │
    └─ 全部失败 ──► 触发 onError 策略
```

### 自愈日志

每次自愈触发时记录详细信息，帮助用户后续优化选择器：

```json
{
  "level": "warn",
  "event": "selector_healed",
  "nodeId": "node_4",
  "originalSelector": "#dynamic-btn-123",
  "healedSelector": "text=Submit",
  "reason": "original selector not found, fallback matched",
  "timestamp": "2026-04-17T12:00:00Z"
}
```
